# 企業諮商個案管理與自動化文件生成系統 (EAP Automation System)

本專案為一套基於 **Google Apps Script (GAS)** 開發的企業諮商行政自動化管理系統。系統旨在解決傳統人工對帳、跨企業個案歷史追蹤、以及繁瑣的 Word 諮商表單代換痛點，達到自動化行政作業。

---

## 🚀 專案目的與核心摘要

在企業諮商實務中，舊個案常因跨年度重複填表、或轉換所屬企業，導致後台數據雜亂。本系統透過**「逆向時間軸搜尋邏輯」**與**「跨企業核對機制」**，確保不論是即時填表、或是手動批次補單，系統抓取到的皆為個案**最新一筆（現況）**的諮商資料。同時，系統會自動跨表進行二線財務/行政註記、智慧動態格式化命檔，並具備後台錯誤警報與業務日報表分權群發機制。

---

## ✨ 核心功能特點

1. **即時填表自動生成 (功能一)**
   - 個案送出 Google 表單後，系統自動觸發，從雲端硬碟複製 Word 諮商表單範本。
   - 自動解析民國年並計算該年度流水號（如 `115001`），自動將表單內如「第一次諮商時間」等所有欄位精準代換至 Word 中。
   - **視覺化行政防錯**：若該企業工作表含有「已提供」欄位則自動打勾；若無該欄位，則自動將試算表該列範圍「整列反灰（淺灰色 1, #f3f3f3）」，維持版面整潔。

2. **跨企業逆向智慧搜尋與再製 (功能二)**
   - 支援手動在「手機批次查詢」分頁輸入單筆或多筆手機號碼。
   - 搜尋邏輯採用「由下往上」倒序掃描，優先抓取最新時間。
   - **跨企業時間爭霸戰**：地毯式掃描 40 多家企業工作表，若同一個案在多家企業皆有歷史紀錄，大腦會進行時間戳記（Timestamp）大比對，強制排除舊資料，僅針對最新的一筆資料生成 Word 文件，並將精準的星期與時間（如：`6/9(二)20:00`）拋接回查詢分頁。

3. **智慧連動動態命名規則**
   - 檔案複製生成時，系統自動連動外部「EAP聯絡總表」。
   - **NCL 標籤偵測**：自動偵測個案最新資料的欄 C 是否含有 `NCL` 文字，動態調整檔名為 `115nnn企業諮商表格(NCL個案姓名)`。
   - **諮商所後綴黏接**：自動偵測欄 F（下拉式選單）是否有指定諮商所，若有，則自動將諮商所名稱黏接於檔名最末端；若無則保持空白。

4. **二線總表智慧自動註記 (功能三)**
   - Word 成功生成後，系統自動跨至全新「EAP聯絡總表」的對應年度分頁（如 `115年`）。
   - 由下往上掃描手機號碼，**優先檢查欄 A（核取方塊）是否為 `false`（未勾選）**。若為 `false` 則於欄 J 自動標記 `V(系統)` 並停止；若為 `true`（代表舊單已結案）則智慧跳過、繼續往上尋找，完美防止重複對帳。

5. **後台警報與業務日報表（分權控管機制）**
   - **資安與後台維護**：當系統發生「找不到年度總表分頁」或「總表中查無此手機無法註記」時，錯誤警告信件僅會單獨寄給**系統維護人員**，以便幕後修正。
   - **業務成果共享**：每日下午 17:00，系統會彙整當日所有成功製作的個案 Log 報表，自動發送業務日報表給**多人名單（包含主管、HR、行政團隊）**，並在發信後自動清空紀錄區。

## 🔒 安全性架構 (SecOps)

為了將程式碼公開至 GitHub 作為開源作品集，本專案嚴格執行**「程式碼與環境變數解耦」**。程式碼中 **100% 找不到任何寫死的雲端資料夾 ID、試算表金鑰或個人 Email**。所有敏感資訊皆鎖在 Google 雲端後台的「腳本屬性」中，具備極高的資訊安全防護。

## 🛠️ 環境準備與部署指南

若欲部署並運行此專案，請依據以下步驟進行設定：

### 1. 設定 Google Apps Script 腳本屬性

請至 GAS 編輯器左側的 **「專案設定 ⚙️」**，捲動至最下方「腳本屬性 (Script Properties)」新增以下五把鑰匙：

| 屬性名稱 (Key)       | 數值內容 (Value)                                                             |
| :------------------- | :--------------------------------------------------------------------------- |
| `TEMPLATE_FOLDER_ID` | 存放公版 Word 範本的 Google 雲端硬碟資料夾 ID                                |
| `TARGET_FOLDER_ID`   | 預計產出個案 Word 結果檔案的目標資料夾 ID                                    |
| `EAP_SHEET_ID`       | 【EAP聯絡總表】試算表的網址 ID                                               |
| `ALERT_EMAIL`        | 系統出錯、總表找不到人時的警報收件人（通常為您自己）                         |
| `SUMMARY_EMAILS`     | 下午 17:00 每日執行成果日報表的群發信箱名單（多個信箱請用半形逗號 `,` 隔開） |

### 2. 試算表分頁結構準備

請確保您的主主控試算表中，已建立好一個名為 **`系統錯誤紀錄`** 的分頁，並在第一列（Row 1）手動建立以下 7 個欄位標題：

- A1: `執行時間` | B1: `企業名稱` | C1: `個案姓名` | D1: `執行狀態` | E1: `訊息 / 失敗原因` | F1: `手機號碼` | G1: `諮商時間`

## 📂 本機開發與版控流程 (clasp)

本專案採用 Google 官方的 `@google/clasp` 工具進行本機開發，並透過 VS Code 與 Git 進行版本管理：

```bash
# 1. 全域安裝 Google Clasp 工具
npm install -g @google/clasp

# 2. 登入 Google 帳號並啟用 Apps Script API
clasp login

# 3. 複製雲端專案至本機資料夾
clasp clone "您的GAS專案ID"

# 4. 本機修改完成後，單次推送到雲端
clasp push

# 5. 開發時開啟自動監聽模式 (儲存檔案即自動 push)
clasp push --watch
```
