# EAP 個案諮商表格自動化系統

本專案是一套部署於 Google Sheets 的 Google Apps Script（GAS），用途是將「Google 表單填寫的個案資料」自動轉換為制式化的 Word 諮商表格，並同步更新公司內部的 EAP 聯絡總表、寄送每日執行報表與預約提醒。

## 系統架構總覽

```
Google 表單（各企業一張表單）
        │  提交
        ▼
Google Sheet（各企業一個分頁，如「1.企業名稱」）
        │  onFormSubmit 觸發
        ▼
onFormSubmitTrigger()
        │  複製範本、代換欄位、核取方塊
        ▼
Google Drive（依諮商日期分類的子資料夾）Word 表格檔案
        │
        ├─→ 回寫本分頁「已提供」欄位 / 整列反灰
        ├─→ 更新 EAP 總表「表格已填」「個案」欄位
        └─→ 寫入「系統錯誤紀錄」分頁
```

另有「批量製作資料」分頁可手動輸入手機號碼，回溯查詢並批次補產表格；以及每日定時任務，於下班前彙整當日執行結果、並預先為明日（或連假前）已預約但尚未填表的個案準備空白表格。

## 事前準備

1. **範本資料夾**：每家企業一份 Word 範本（`.docx` 以 Google 文件形式存放），檔名需與 Google Sheet 的分頁名稱完全一致（例如分頁叫「1.某A企業」，範本檔名也要叫「1.某A企業」）。
2. **目標資料夾**：存放產出 Word 表格的資料夾，系統會依「諮商日期（m/d）」自動建立子資料夾分類存放。
3. **EAP 聯絡總表**：另一份獨立的 Google Sheet，依年度（民國年，如「115年」）分頁記錄個案聯絡與進度狀態。
4. 在同一份主 Sheet 中建立「系統錯誤紀錄」與「批量製作資料」兩個分頁（詳見下方欄位需求）。
5. 在 Apps Script 專案的 **Script Properties** 中填入必要的 ID 與 Email（詳見下方章節）。

## 功能模組說明

### 模組一：表單提交自動製表

**主函式：`onFormSubmitTrigger(e)`**

當任一企業的 Google 表單被提交、資料寫入對應分頁時觸發，自動執行：

1. **讀取表單欄位**：姓名、性別、身分證字號、出生年月日、教育狀況、婚姻狀況、電話、地址、緊急聯絡人資訊、問題陳述、資訊來源、第一次諮商時間，以及依企業客製的彈性欄位（員工編號、子公司、工作縣市、廠區、身分別）。
2. **格式化處理**：
   - 出生年月日轉為 `yyyy/m/d`。
   - 諮商時間解析「上午/下午」中文時間格式，轉為 `m/d(星期) HH:mm`（24 小時制）。
   - 手機號碼自動格式化為 `09XX-XXX-XXX`。
3. **產生流水編號**：以時間戳記推算民國年 + 該年度累計筆數，格式為 `民國年 + 3 碼流水號`（例如 `115001`），並以 `LockService` 避免併發衝突。
4. **查詢 EAP 總表**：比對手機號碼，判斷是否為 NCL 個案、所屬諮商所地點，作為檔名前綴/後綴。
5. **依諮商日期建立子資料夾**，若無法解析日期則存放於根目錄並寄信通知管理者人工確認。
6. **複製範本並產生 Word 檔**：檔名格式為
   `流水號 + 企業名稱 + 諮商表格(NCL可選+姓名) + 諮商所名稱`
7. **文字與核取方塊代換**：將範本中的 `{{欄位名稱}}` 佔位符換成實際內容或勾選符號 `■ / □`（詳見下方對照表），支援單選與複選（含「其他」自填文字）。
8. **回寫原分頁**：若有「已提供」欄位則勾選，若無則將整列背景刷成淺灰色標記已處理。
9. **同步 EAP 總表**：呼叫 `updateYearlyChecklist` 與 `updateCaseNameInEapSheet`。
10. **寫入執行紀錄**：呼叫 `logStatus` 記錄成功或失敗原因至「系統錯誤紀錄」分頁。

**輔助函式：`logStatus(company, name, status, msg, phone, time)`**
將每次執行結果（成功/失敗、原因、手機、諮商時間）附加寫入「系統錯誤紀錄」分頁，供每日彙整信件使用。

**每日報表：`sendDailyErrorSummaryEmail()`**
建議設定為每日 17:00 執行，讀取「系統錯誤紀錄」分頁，彙整當日所有成功與失敗紀錄寄送 Email 給管理者，寄出後自動清空紀錄（保留標題列）。

### 模組二：個案資料批量查詢與再製

用於：表單提交當下漏產、系統故障後補產、或需依手機號碼回溯查找歷史個案時使用。

**`batchSearchAndGenerateWord()`** — 選單「有資料才製作」

1. 讀取「批量製作資料」分頁 A 欄的手機號碼清單。
2. 跨所有企業分頁比對電話（去除格式符號後比對），若同一人在多家企業都有填寫，取「時間戳記最新」的一筆為準。
3. 找到資料則透過 `createMockEvent` 模擬表單提交事件，呼叫 `onFormSubmitTrigger` 產出完整版 Word。
4. 找不到資料則於 B 欄標示「❌ 查無資料」。
5. 執行結果（訊息、諮商時間、時間戳記）寫回 B〜D 欄。

**`batchSearchAndGenerateAllWord()`** — 選單「全部製作（無資料產空白版）」

- 找到表單資料的邏輯與上述相同，產出完整版。
- **找不到表單資料**時，改查詢 EAP 總表取得個案姓名、地點、公司名稱，透過 `_findTemplateByCompany` 比對範本、`_replaceAllPlaceholdersWithBlank` 將所有佔位符清空/取消勾選，產出「空白版」Word 表格，並回寫 EAP 總表「表格已填」欄為「給空白(系統)」。
- 完全查無資料（EAP 也沒有或公司欄空白）則標示查無資料或找不到範本。

**輔助函式：`createMockEvent(sheet, row, headers)`**
將指定分頁、指定列的資料，包裝成與真實表單提交事件（`e.namedValues`）相同結構的模擬物件，讓 `onFormSubmitTrigger` 可重複使用同一套邏輯。

### 模組三：EAP 總表自動註記

**`updateYearlyChecklist(phone, twYear)`**
Word 表格產製完成後，回到 EAP 聯絡總表對應年度分頁，依手機號碼找到「已完成＝未勾選」的最新一筆，將「表格已填」欄註記為 `V(系統)`；若找不到，改找含「給空白」字樣的補交紀錄，註記為 `已補交(系統)`。兩輪都找不到則寄信通知管理者人工核對。

**`updateCaseNameInEapSheet(phone, fullName, twYear, gender)`**
將 EAP 總表「個案」欄的簡短代稱（如「王生」）依規則補齊為全名＋稱謂（如「王大明先生」），並保留原有的 NCL 前綴；若原欄位沒有稱呼且非 NCL 個案則不處理。含詳細 `Logger.log` 除錯紀錄。

### 新功能：預約提醒自動產生空白表

**`sendDailyUpcomingCaseReminder()`**

建議設定為每日 17:00 時間驅動觸發，依當天星期決定要處理的日期範圍：

| 觸發當天   | 產製對象日期                  |
| ---------- | ----------------------------- |
| 週一〜週四 | 隔天 1 天                     |
| 週五       | 週六、週日、下週一（共 3 天） |
| 週六、週日 | 不執行                        |

執行邏輯：

1. 開啟 EAP 總表對應年度分頁，讀取「下次約的日期」「已完成」「表格已填」「公司」「個案」「地點」欄位。
2. 篩選出：日期符合目標日、尚未完成、且「表格已填」欄未含 `V` 或「給空白」的個案。
3. 依公司名稱找範本、依諮商日期建立/取用子資料夾。
4. 產生「全空白」版本 Word（所有佔位符清空或取消勾選），檔名加註「-空」。
5. 回寫 EAP 總表「表格已填」為 `給空白(系統)`。
6. 彙整成功與失敗清單，寄信通知管理者（需設定 `SUMMARY_EMAILS`）。

> ⚠️ 國定假日不在此自動邏輯判斷範圍內，遇連續假期請改用「批量製作資料」選單手動處理。

### 輔助工具：手動選單製作空白表

**`showCompanySelector()`** + **`generateBlankTemplateByCompany(companyName)`**
提供一個簡易下拉選單視窗，可手動選擇任一企業，直接產出「純空白」Word 表格（不綁定任何個案資料），存放於目標資料夾根目錄，供臨櫃或現場緊急使用。

## 試算表欄位需求

### 主 Sheet（各企業表單分頁，如「1.企業名稱」）

標題列須包含（欄名需與表單問題完全一致）：
`時間戳記`、`姓名`、`生理性別`、`身份證字號`、`出生年月日`、`教育狀況`、`婚姻狀況`、`聯絡電話(手機)`、`聯絡地址`、`緊急聯絡人姓名`、`緊急聯絡人電話`、`緊急聯絡人關係`、`問題陳述(可複選)`、`資訊來源(可複選)`、`第一次諮商時間`，以及選用的彈性欄位（`員工編號`、`隸屬子公司`、`工作縣市`、`所屬廠區`、`身分別`）與選用的 `已提供` 欄位。

### 「系統錯誤紀錄」分頁

7 欄，依序：執行時間、企業、姓名、狀態（成功/失敗）、訊息、手機、諮商時間。

### 「批量製作資料」分頁

A 欄輸入手機號碼，B〜D 欄由系統自動寫回（結果訊息、時間戳記、諮商時間）。

### EAP 聯絡總表（獨立試算表）

依民國年分頁（如「115年」），需包含：`手機`或`電話`、`個案`、`地點`、`已完成`（核取方塊）、`表格已填`、`公司`、`下次約的日期`。

## 錯誤處理與通知機制

- 找不到範本、資料夾 ID 錯誤、EAP 總表欄位缺漏等狀況，皆會寫入「系統錯誤紀錄」分頁並於每日報表中呈現，部分關鍵錯誤（找不到年度工作表、欄位缺漏、資料夾權限問題）會即時寄信至 `ALERT_EMAIL`。
- 找不到諮商日期無法歸檔時，Word 檔會存放於根資料夾並即時寄信通知。
- EAP 總表比對後找不到可註記的個案時，會即時寄信提醒人工核對。
- 所有跨企業比對邏輯皆以「電話號碼去除非數字字元」後比對，避免格式不一致造成漏判。

## 常見問題

**Q：同一人在多家企業都填過表單，系統會用哪一筆？**
比對電話號碼後，一律取「時間戳記最新」的一筆資料製表。

**Q：範本檔名一定要跟分頁名稱一模一樣嗎？**
`onFormSubmitTrigger` 是精確比對分頁名稱；而空白版相關功能（`_findTemplateByCompany`）則採包含比對（去除數字編號後互相包含即可），彈性較大。

**Q：流水號規則是什麼？**
`民國年（3碼）+ 該年度累計第幾筆（3碼補零）`，例如 115 年第 8 筆為 `115008`，每年自動歸零重新計算。

**Q：如何避免同一個案被重複產製？**
一般表單流程只在提交當下觸發一次；批量／預約提醒流程則透過檢查「已提供」欄位、EAP 總表「表格已填」欄位（含 `V` 或「給空白」字樣）來避免重複產製。

## 本機開發與版控流程 (clasp)

本專案採用 Google 官方的 `@google/clasp` 工具進行本機開發，並透過 VS Code 與 Git 進行版本管理：

```bash
# 1. 全域安裝 Google Clasp 工具
npm install -g @google/clasp

# 2. 登入 Google 帳號並啟用 Apps Script API
clasp login

# 3. 複製雲端專案至本機資料夾
clasp clone "您的GAS專案ID"

# 4. 本機修改完成後，單次推送到雲端
clasp push

# 5. 開發時開啟自動監聽模式 (儲存檔案即自動 push)
clasp push --watch
```
